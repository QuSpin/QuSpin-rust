## Comment #10 (commit hygiene)

Commit `d5934bf` ("refactor(workspace): decouple quspin-basis from
quspin-operator via StateGraph") contains a mid-sentence correction in
the message body: *"Five PyO3 binding files (operator/bond, …) —
actually here the updates are in basis/\*.rs"*. The correction is
right but the presentation is sloppy — a reviewer should not have to
read a retraction in a commit message.

That commit is already pushed as part of PR #57, and rewriting
published history on a branch in review is disruptive for the
reviewer. The entry lives in the log as a known bit of untidiness; the
accurate summary is in `CHANGELOG.md` (commit `8e9965e`), which
reviewers should trust over the original commit message.

Lesson filed: run `git log -1 --format=%B` against every new commit
before push, especially when generating messages from a checklist.
